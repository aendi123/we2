Media Queries anwenden.

Nutzen Sie ihr Layout aus der Aufgabe "Flexible Layout > Layout erstellen"

Dieses Layout soll für Smartphones wie folgt aussehen. Nutzen Sie dafür Media-Queries.

  • 1) Desktop First Design d.h. zusätzliches CSS in einer Query die nur bei Mobile-Screens aktiv ist
  • 2) Mobile First Design mit Flexbox d.h. das Basis CSS ist für Mobile-Screens optimiert. Bei breiteren Screens wird das zusätzliche CSS aktiv
  • 3) Mobile First mit Progressive Enhancement Design: Basis-Design: Mobile Design ohne Flexbox d.h. das Basis-CSS ist für Mobile-Screens und Screens ohne Flexbox-Support optimiert. Bei breiteren Screens wird das zusätzliche CSS mit Flexbox aktiv. (optional: enstprechende @supports query)

  • 1) Desktop First mit Flexbox bei Desktop und Mobile
  • 2) Mobile First mit Flexbox bei Mobile und Desktop
  • 3a) Mobile First Progressive Enhanced: Mobile = Box-Layout, Desktop=Flexbox. Vertikale Zentrierung von "Center" mittels line-height.
  • 3b) V2: Mobile First Progressive Enhanced: Mobile = Box-Layout Desktop=Flexbox. Vertikale Zentrierung von "Center" mittels ::before block. Nutzt @supports. Für Test, dass @supports funktioniert, eigent sich das Chrome Plugin CSS Feature Toggles

Ist Ihre Lösung / bzw. die Muster Lösung Mobile First? Begründen Sie.

Nein - Begründung: Erst durch die Media Queries wird die Mobile Ansicht sichtbar. Dies wiederspricht Mobile First - Es könnte sein, das Media Queries nicht unterstützt werden wiederspricht dem "Progressive Enhancement"-Prinzip.

Container Queries (1)

Die Container Query im folgenden File container-ex-1.html "funkioniert nicht"

Aufgabe: Das Problem lösen

Das erwartete Verhalten wäre, dass das .demo-container Div vom Nutzer grösser gemacht werden kann (resize: horizontal; overflow: hidden;) Ab der Grösse 500px sollte dann durch die container query (@container) das .styled-child div die Farbe und den Text ändern. Hierbei spielt keine Rolle, dass das .styled-child kein direktes Kind von .demo-container ist. Das Problem ist, dass bei .demo-container container-type: inline-size nicht gesetzt ist. Container Queries funkionieren nur wenn container-type auf inline-size oder size gesetzt ist. In diesem File container-ex-1-solution.html ist es korrigiert

Container Queries (2)

Die Container Query im folgenden File container-ex-2.html "funkioniert nicht"

Aufgabe: Das Problem lösen

Das erwartete Verhalten wäre, dass das .demo-container Div vom Nutzer in der Höhe grösser gemacht werden kann (resize: vertical; overflow: hidden;). Ab der Höhe von 300px sollte dann durch die container query (@container) das .styled-child div die Farbe und den Text ändern. Das Problem ist, dass bei .demo-container container-type: inline-size gesetzt ist. Dies erlaubt keine Container-Query welche die Höhe abfragt (nur width bzw inline-size). In diesem File container-ex-2-solution.html ist es korrigiert
